# 快速开始

本文档将帮助您快速创建并运行第一个星火编辑器2.0项目。

## 新建项目

2.0版本的编辑器在新建项目时提供两种模板选择：

### 空模板
创建一个完全空白的项目，没有任何示例代码。适合有明确开发目标的资深开发者。

### 带示例的模板
创建一个包含多个示例项目的完整模板。

在示例模板中：
- 您可以通过修改 `GlobalConfig.cs` 文件来选择要运行的示例项目
- 所有可选的示例项目都在 `GlobalConfig.cs` 中列出
- 每个示例都展示了框架的不同特性和用法

## 编辑器功能

### 图形化编辑器

在图形化编辑器中，您可以像以前一样进行：
- **地形编辑** - 创建和编辑游戏地图
- **玩家队伍设置** - 配置玩家和队伍
- **项目设置** - 配置基础的项目参数

⚠️ **当前限制**：触发编辑器和数据编辑器暂未开放（内部开发中）。

## 文档和API列表

### 通过解决方案管理器访问

在编辑器中新建项目后，对应的C#解决方案项目根目录下会有一个**Docs目录链接**。

**在Visual Studio或Rider中**：
- 打开解决方案后，可以直接在解决方案管理器中看到Docs链接
- 点击即可浏览文档和API列表

**在VSCode或Cursor中**：
- 需要在 `.sln` 文件上右键，选择"打开解决方案"
- 然后在解决方案管理器(Solution Explorer)中才能看到Docs链接

### 手动查找文档目录

如果无法通过解决方案管理器访问，可以手动定位文档：

1. 打开项目的 `src/WasiCoreSDK.props` 文件
2. 查看 `WasiCoreSDKPath` 属性，它定义了SDK的实际安装路径：

```xml
<!-- src/WasiCoreSDK.props -->
<Project>
  <PropertyGroup>
    <WasiCoreSDKPath>SDK的实际路径</WasiCoreSDKPath>
  </PropertyGroup>
</Project>
```

3. SDK路径会随安装目录变化，也会随着编辑器的更新而变化，请始终从该文件读取实际路径。要刷新最新的SDK路径，请在编辑器中打开一次项目，然后再次查看该文件。
4. API文档和通用文档都在该路径下的 `api` 和 `docs` 目录中

## 构建项目

2.0版本使用.NET 9.0，构建时需要指定配置标志。

### 编译服务端调试版本

```bash
dotnet build -c Server-Debug
```

### 编译客户端调试版本

```bash
dotnet build -c Client-Debug
```

如果有编译错误，请修正后再次编译验证。
在编辑器中点击调试按钮，编辑器会自动编译这两个版本。因此这里列出的构建命令仅供参考。
